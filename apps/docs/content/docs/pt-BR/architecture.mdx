# Arquitetura do Gateway

O gateway é construído com uma arquitetura de interface dupla: expõe uma **API REST** tradicional (documentada via OpenAPI/Swagger) e uma **API GraphQL** flexível no mesmo servidor, permitindo que clientes escolham a interface que melhor se adequa aos seus padrões de integração.

## Escolha GraphQL vs REST

### Quando escolher REST
- **Simplicidade**: Melhor ao integrar com cURL padrão, handlers de webhook ou ferramentas de automação de fluxo de trabalho de terceiros (ex: Make, Zapier, n8n).
- **Playground Interativo**: Ótimo para testar operações específicas dentro da nossa página sandbox Swagger UI integrada.
- **Saída Previsível**: Endpoints padrão retornam schemas fixos, ideal para mapeamentos estáticos de cliente.

### Quando escolher GraphQL
- **Seleção de Dados**: Consultar apenas os campos que você precisa (ex: buscar apenas títulos de vagas para minimizar tamanho de resposta).
- **Execução em Endpoint Único**: Consolidar múltiplas buscas de recursos (ex: buscar info do perfil e listar vagas correspondentes em uma única requisição).
- **Geração Dinâmica de Cliente**: Gerar interfaces de cliente TypeScript automaticamente usando ferramentas como GraphQL Codegen.
