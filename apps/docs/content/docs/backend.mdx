# JobFinder Main Backend

The **JobFinder Main Backend** is the core orchestrator of the LinkedIn job search and application automation toolchain. It manages user sessions, tracks application logs, handles resume formatting, and calls large language models (LLMs) to automatically optimize credentials and generate questionnaire answers.

The backend acts as the bridge between:
1. **Frontend User Interface**: Expresses user options and renders search results and application histories.
2. **JobFinder Sync Extension**: Cache endpoint for cookies and profiles.
3. **LinkedIn Gateway Service**: Triggers job search, forms extraction, and application submission on Voyager.
4. **9Router / LLM Provider**: Automates resume tailored parsing and generates Easy-Apply responses.

---

## Technical Stack

- **Framework**: Express (Node.js) written in TypeScript.
- **ORM & Database**: Prisma with SQLite (`better-sqlite3` driver).
- **LLM Client**: Integrates via standard Open-AI compatible headers to communicate with Gemini / Claude models.
- **PDF Compiler**: `pdfkit` for programmatic resume PDF builds.
- **Browser Automation**: `puppeteer` for headless background tasks.

## Database Models

The SQLite database schema defines four main models (defined in [schema.prisma](file:///Users/macbookpro/Documents/linkedin-job-explorer/apps/linkedin-job-backend/prisma/schema.prisma)):

1. **Credentials (`credentials`)**: Stores active session `cookie`, `csrf` token, and custom voyager environment request headers.
2. **Resume (`resumes`)**: Stores parsed LinkedIn profiles, including personal bio, name, headline, experience details, and education entries as structured JSON.
3. **Application (`applications`)**: Tracks Easy-Apply operations. Contains status (`draft`, `applied`, `failed`), questions answers, and optimized resume PDF representation.
4. **Post (`posts`)**: Stores feed updates, publication schedules, and engagement counters (likes, comments, views).

## Core Routing Modules

Endpoints are organized inside `src/routes/` and expose the following controls:

### 1. Credentials Controller (`/api/credentials`)
Receives session cookies and user-agent settings harvested by browser extension popup.
- `GET /status`: Verifies if valid credentials are cached.
- `POST /`: Saves browser session cookies and headers.

### 2. Jobs Controller (`/api/jobs`)
Direct proxy to the Gateway Service to parse easy-apply jobs.
- `GET /`: Lists LinkedIn easy-apply job postings using keywords.
- `GET /:id`: Retrieves full company information and job descriptions.
- `GET /:id/apply-form`: Parses easy-apply form elements (input fields, checkboxes).
- `POST /:id/apply`: Tailors and submits responses with attached optimized resume.

### 3. Resume Controller (`/api/resume`)
Formats profile datasets.
- `POST /from-linkedin`: Sincronizes profile elements to local SQLite database.
- `POST /optimize`: Programmatically generates optimized text summaries based on a target job description.
- `GET /download`: Compiles customized resume PDF binary on the fly using `pdfkit`.

### 4. AI Controller (`/api/generate-answers`)
Invokes LLM API to automate application question answering:
- `POST /api/generate-answers`: Analyzes the form questions URNs alongside candidates' profiles and compiles answers dynamically.
